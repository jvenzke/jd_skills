Introduction

Building AI Workflows

  • Aim for 2-3x Speed (from manual): A well-designed AI workflow accelerates planning and implementation, significantly speeding up your day-to-day work.
  • Keep Context < 100K Tokens: Model performance degrades with overly large contexts. Keep context tight and focused.
    Smart zone Dumb zone
  • Iterate and Adapt: These workflows aren't static. Improve and adjust them over time to perfectly meet your specific needs.

What are Cursor Skills?

Skills enforce consistency when interacting with the agent. By defining operating procedures in Markdown, you guide the AI to follow predictable steps and write artifacts to disk.

Invocation: Skills can be triggered manually (for example slash commands like /grill-me) or automatically by the agent when your request matches a skill’s description.

  • Reduce boilerplate: Use structured workflows to handle repetitive data pipeline setup.
  • Structured prototyping: Keep the AI focused on one task at a time when building Streamlit apps, Snowflake queries, etc.
  • Manage context: Automatically have the agent rely on generated markdown artifacts (e.g. LIGHT_DEV_PLAN.md) instead of trying to remember the whole chat.

The Engineer's Responsibility

AI is an accelerator, not an autopilot. Your role shifts from typing every line to orchestrating the work.

You are the Owner of the Work

  • Accelerate, don't delegate: Use AI to drastically speed up planning and implementation, but remember that you remain the architect, engineer, scientist, and reviewer of the work.
  • Read the code: If the AI generates code that will live in production, you must read and understand it. Treat the AI like a junior developer submitting a PR.
  • QA everything: Rigorously test and review all outputs to ensure the agent actually accomplished what you asked for without introducing regressions.
  • Iterate, don't fix: If the agent makes a mistake, iterate on the plan, not the code. The agent should be able to fix its own mistakes.
  • Customize the workflow: My workflow maybe different than how you work. Iterate on the workflow to make it work for you.

Human in the Loop

Token Efficiency & Spend

  • Efficiency: Human-in-the-loop development using tools like /grill-me is significantly more token efficient for larger tasks than letting an agent run autonomously for long periods.
  • Strategic Usage: Limit automated workflow usage to light tasks or tasks you run in parallel. Avoid unneeded spend on large, poorly-defined tasks.
  • Stay Engaged: Use the agent to accelerate your thinking, not replace it. Frequent alignment prevents the agent from going down expensive rabbit holes.

Cursor Tools & Features

Tools, Rules, Skills & MCPs

  • AskQuestion Tool: Ask the agent to use the AskQuestion tool when it needs clarification. This provides a structured way to provide feedback.
  • Canvases & Notebooks: Use Cursor Canvases or Jupyter Notebooks for interactive graphics, plots, and data exploration.
  • TODO List: Ask the agent to maintain a TODO list to ensure it follows every step of a complex workflow without skipping.
  • Semantic Search: Use SemanticSearch for broad codebase exploration and finding relevant logic by meaning rather than just keywords.
  • Rules: Persistent guidance you want applied in (almost) every chat—coding standards, repo conventions, safety rails. Keep the rule set small and focused; overly long rules add context cost on every session.
  • Skills: Procedures you invoke on demand—manually or when a situation matches—often as slash commands. They can encode multi-phase plans (plan → do → review) without loading that bulk into every conversation.
  • MCPs (Model Context Protocol): Connect the agent to external systems—databases, APIs, ticketing, docs—via MCP servers you enable in Cursor. Lets the agent call approved tools and pull live context instead of pasting everything into chat.
  • Subagents: Use the Task tool to launch subagents for deep-dive explorations or parallel tasks. You can specify a specific model for a subagent to optimize for speed or reasoning.

Workflow Overview

Research

  • Light Research: Plan-then-execute loop for taking an idea end-to-end to a minimal prototype with fast iterations.
  • Linear Research: Focused on detailed user input plans and iteration to drive research toward specific outcomes.

Development

  • Light Dev: Plan-then-execute loop for well-scoped hotfixes and small tasks.
  • Full Dev: Artifact-driven process (PRD → issue files → TDD) for major features and refactors. Supports parallel execution.

Review & General

  • PR Review: Multi-phase review workflow (brief → security → tests → logic) for thorough GitHub PR analysis.
  • General Skills:
    • /grill-me: Relentless interviewing to reach shared understanding.
    • /meta-review: Self-evaluation of skill performance to drive continuous improvement.

Planning & Token Management

  • Plan Quality: The agent is only as good as the plan you provide. Be detailed with your asks and proactive in redirecting the agent during reviews.
  • Token Awareness: Be mindful of token usage; large tasks can consume significant resources. Use concise context and clear handoffs.

Working Effectively with AI

1. Manage Context & Large Data

  • Keep it lean: Dumping raw CSVs or large HTML files into chat degrades agent performance and consumes excessive tokens.
  • Summarize and sample: Use df.describe(), schema definitions, or head/tail samples instead of full files.
  • Route bulk through Python: Have the agent use Python to write large generated outputs and to consume large query/CSV results from disk—rather than reading or pasting them through the chat.
  • Subagents for exploration: Use subagents for broad codebase search or running tests so they return a concise summary instead of flooding your chat with full tool output.

2. Model Selection

  • Start with Auto: Use Cursor’s Auto model selection until you hit quality gaps, stubborn bugs, or tasks that clearly need more reasoning.
  • Upgrade when needed: Switch to a stronger or specialized model only when Auto isn’t enough—saves cost and keeps defaults simple.

3. Explicit Handoffs

  • Write to files: Save plans and intermediate results to disk.
  • Don't rely on chat memory: A new session should be able to resume work from saved artifacts (e.g. LIGHT_DEV_PLAN.md).

4. Leverage MCPs (Model Context Protocol)

  • Extend agent capabilities: MCPs allow the agent to securely interact with external tools, databases, and APIs without you writing boilerplate code.
  • Keep context fresh: Instead of pasting static data, let the agent pull live context exactly when it needs it.
  • Limit active MCPs: Only enable the MCPs you actively use. Having too many enabled will cause context bloat at the start of every chat.

5. Snowflake Guidelines

  • Use the MCP: Look up schemas and pull small samples (< 50 rows).
  • Check sizes: Verify table sizes before querying to avoid timeouts.
  • Full results: Same bulk-data rule as above—use the Python connector to land full datasets in files and analyze there, not through chat.

Demo: Linear Research Workflow

The Goal

  • Pull the California Housing dataset.
  • Test multiple prediction methods.
  • Perform hyperparameter tuning.
  • Rank models and report sensitivity.

The Prompt

# California Housing

I would like to build a predictor for the California Housing dataset available via sklearn 

Basic goal
- Develop a tool to pull the dataset and make predictions
- Try out a handful of methods
- Do hyperparameter tuning
- Provide a best solution for this dataset 
- Report the results as a ranking

Skill calls

Run the linear research skills in order; repeat /r-plan → /r-do until the work is done, then close out.

  1. /r-setup — Create the research project layout and stubs (input_plan.md, research_plan.md, research_log.md, next_step.md, Canvases/, src/). Fill in input_plan.md with your goals (e.g. the California Housing prompt above), then continue.
  2. /r-plan — From input_plan.md, define the next research step only and update planning artifacts / next_step.md.
  3. /r-do — Execute what’s in next_step.md (code, plots, verification canvases). Save each .canvas.tsx under Canvases/ in the project **and** the Cursor IDE canvases/ path so it opens beside chat. Loops back to /r-plan for the next slice, or forward when ready to summarize.
  4. /r-summarize — Consolidate outcomes into the project README.md with findings, artifacts, and handoff notes.
